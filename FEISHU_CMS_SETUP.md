# 仙人森林 · 飞书表格后台 配置指南

产品数据由「飞书表格」驱动。你日常只在飞书里改表格；网站在**用户每次访问时**通过 Cloudflare Pages Functions **实时**从飞书读取最新数据，无需重新部署、无需定时任务。

---

## 一、整体流程

```
你在飞书表格里改产品
        │
        ▼  保存（飞书自动同步到云端）
访客打开网站 → /api/products 被调用
        │
        ▼  Cloudflare Function 实时拉取飞书表格 → 转成产品 JSON（CDN 缓存 60 秒）
网站显示最新产品
```

你（非技术）每天只做一件事：**打开飞书表格，改内容，保存，关掉。** 下一次有人访问网站（最多 60 秒延迟）就会看到更新。没有"部署""同步"这些额外步骤。

> 飞书拉取失败时，网站自动降级使用仓库里内置的 `products.json`，**永远不会因为表格出错而白屏**。

## 二、一次性设置（只需做一次）

### 第 1 步：在飞书开放平台创建自建应用

1. 打开 https://open.feishu.cn （飞书开放平台），登录你的企业账号。
2. 进入 **开发者后台 → 创建应用 → 企业自建应用**，起个名（如 `仙人森林产品同步`）。
3. 创建后进入应用详情，记下 **App ID** 和 **App Secret**（在「凭证与基础信息」里）。
4. 左侧「权限管理」→ 搜索并开通 **`sheets:spreadsheet:readonly`**（读电子表格，只读即可）。
5. 左侧「版本管理与发布」→ **创建版本 → 填写信息 → 发布**。
   - 发布后，这个应用才有权限访问你企业内的文档。

### 第 2 步：建表格并共享给应用

1. 在飞书里新建一个**电子表格**（不是多维表格），命名为「仙人森林产品库」。
2. 把表头按下方「列说明」填好（或直接导入本项目里的 `docs/feishu_template.csv`）。
3. 在飞书**桌面端**打开这个云文档，点右上角 **「分享」→「添加协作者」**，在搜索框里搜你刚建的应用名（它会以「机器人」身份出现），选中后权限设为 **「可阅读」**，确认。（这就是"把表格共享给应用"，应用才能读它。手机端可能没有"添加协作者"入口，请用电脑。）
4. 打开这个表格，从地址栏复制 **表格 Token**：URL 里 `sheets/` 后面那一长串，如
   `https://...feishu.cn/sheets/【这就是 token】/???sheet=...`

> 工作表范围（sheetId / 读取区间）**不用填**——函数会自动探测第一个工作表并读取 `A1:Z1000`。

### 第 3 步：把密钥填进 Cloudflare（不是 GitHub）

1. 打开 Cloudflare 控制台 → 你的 Pages 项目 `immortal-forest-site` → **Settings → Variables and secrets**。
2. 点 **「+ Add」**，依次添加：

| 变量名 | 必填 | 值 | 说明 |
|------|------|------|------|
| `FEISHU_APP_ID` | ✅ | 第 1 步的 App ID | 选 **Text** 类型 |
| `FEISHU_APP_SECRET` | ✅ | 第 1 步的 App Secret | 选 **Secret** 类型 |
| `FEISHU_SHEET_TOKEN` | ✅ | 第 2 步的表格 Token | 选 **Secret** 类型 |
| `FEISHU_SHEET_RANGE` | ⬜ | 例如 `Sheet1!A1:K1000` | 留空即可（自动探测） |
| `SITE_URL` | ⬜ | `https://immortal-forest-site.pages.dev` | 用于解析相对图片路径 |

3. 保存后，回到 **Deployments** 点 **「Retry deployment」**（或等下次提交）重新部署一次，变量才会生效。

> 变量名兼容写法：代码同时认 `FEISHU_SHEET_TOKEN` 和 `FEISHU_SPREADSHEET_TOKEN`，填哪个都行。

## 三、表格列说明（必须一一对应）

第一行是表头，顺序随意，函数按**列名**匹配（中英文都认）。推荐用中文列名：

| 列名 | 含义 | 示例 | 对应网站字段 |
|------|------|------|------|
| 分类 | 产品类别 key | `stationery` / `device` / `furniture` / `digital` / `consumable` | `cat` |
| 图片 | 图片网址（见第四节） | `https://immortal-forest-site.pages.dev/images/p1.jpg` | `img` |
| 中文名 | 中文名称 | `森呼吸中性笔` | `zh.name` |
| 中文描述 | 中文简介 | `竹纤笔身、速干墨芯……` | `zh.desc` |
| 中文价 | 中文价格 | `¥ 3.9 / 支` | `zh.price` |
| 英文名 | 英文名称 | `Forest-Breath Gel Pen` | `en.name` |
| 英文描述 | 英文简介 | `Bamboo barrel, quick-dry ink...` | `en.desc` |
| 英文价 | 英文价格 | `from $0.12 / pc` | `en.price` |
| MOQ | 最小起订量 | `5,000 pcs` | `spec.moq` |
| 交期 | 交货周期 | `7–15 days` | `spec.lead` |
| 贸易术语 | 贸易条款 | `FOB Shanghai` | `spec.term` |

- 留空的单元格 = 空字符串（网站会自动用图标占位代替缺图）。
- 新增产品：在表格末尾加一行。删除：删掉那一行。调整顺序：直接拖行。

## 四、图片怎么放（免费方案：存 GitHub 仓库）

图片**不进飞书**，只把「网址」填进「图片」列。免费且最快的做法：

1. 在你的 GitHub 仓库里，进 `images/` 文件夹（没有就新建），点 **Add file → Upload files** 把产品图传上去。
2. 传完后，图片的访问网址是：
   `https://immortal-forest-site.pages.dev/images/你的文件名.jpg`
3. 把这个网址整段复制，粘到飞书表格的「图片」列。

> 也可以只填相对路径 `images/p1.jpg`，函数会自动拼上 `SITE_URL` 变成完整网址。
> 图片建议压到每张 100KB 以内（宽 800–1200px 足够），加载更轻快。

## 五、怎么验证 / 改完多久生效

- **改完多久生效**：保存表格后，下次访问网站最多延迟 **60 秒**（CDN 缓存）即更新，无需任何手动操作。
- **快速验证飞书是否连通**：浏览器打开
  `https://immortal-forest-site.pages.dev/api/products?debug`
  - 返回 `"source":"feishu"` → 飞书已连通，改表格网站会实时变 ✅
  - 返回 `"source":"fallback"` 且带 `error` → 还没连上，看 `error` 提示缺什么（通常是密钥填错或表格没共享给应用）
- **看真实产品数据**：`https://immortal-forest-site.pages.dev/api/products`

## 六、回滚 / 应急

- 如果飞书表格填错导致网站异常：函数会自动降级使用仓库内置的 `products.json`，网站照常运行、不会崩。修正表格后 60 秒内自动恢复。
- 想强制某版本产品：直接编辑仓库里的 `products.json` 并提交，网站会把它作为兜底数据使用。
- 原来的 `/admin/` Decap 后台已彻底移除，不要再使用；以飞书表格为准。

## 七、常见问题

**Q：改表格要花钱吗？**
A：Cloudflare Pages Functions 和飞书自建应用都免费，零成本。

**Q：只想自己改，不想让应用有我的文档权限？**
A：应用仅被授权读那一个共享出去的表格，看不到你其他文档，安全可控。

**Q：能多人同时编辑吗？**
A：可以，飞书表格本身支持协作，谁改都行，网站实时反映。

**Q：在「添加协作者」里搜不到我建的应用？**
A：99% 是因为应用**没发布**。创建应用 ≠ 发布应用。必须去飞书开放平台 → 你的应用 → 左侧「版本管理与发布」→「创建版本」→ 填版本号（如 1.0.0）和说明 →「发布」。发布后应用才变成可被文档搜索到的"机器人"（可能延迟几分钟）。若发布后显示"审核中"，等它过（自建应用通常秒过；若企业要求管理员审批，去审批一下）。发布成功的话，在飞书客户端左侧「工作台」能搜到这个应用名。

**Q：发布后还是搜不到？**
A：检查应用「可用范围 / 可见范围」是否包含你自己；部分企业要求管理员在「管理后台」手动启用该自建应用后，它才会出现在协作者搜索里。
